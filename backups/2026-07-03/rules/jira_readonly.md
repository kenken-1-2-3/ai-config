# Jira Read-Only Rules

Apply these rules whenever Jira, Atlassian, or Rovo Jira tooling is available.

- Jira access is read-only by default.
- You may search, inspect, query, summarize, and link to Jira issues, projects, boards, filters, and related metadata.
- Do not create, edit, delete, transition, assign, comment on, link, unlink, log work on, or otherwise mutate Jira issues or projects.
- Do not make Jira writes even if a connector or MCP tool exposes write-capable operations.
- If a task appears to require a Jira write, explain what would need to change and ask the user to perform or explicitly authorize that specific write action first.
